Product Personality

Principles for the whole surface

01

Show provenance

Every design doc should expose what fed it, what mutated it, and how far it drifted from the seed.

02

Let distance mean something

The 2D plane is not decorative. Spatial relationships carry meaning about contamination, kinship, and branch depth.

03

Reward browsing

The gallery should feel archival and exploratory, with pinned branches, comparison states, and re-entry into old storms.

04

Stay legible under weirdness

The product earns trust by making unusual results inspectable instead of mysterious.

Interaction Grammar

Visual vocabulary for the product

These are the recurring interface moves that should define Design Storm across the canvas, docs, and gallery.

Node Type

Seeds

Heavier cards with calm glow. They anchor the map and remain stable while the storm forms around them.

Node Type

Entropy sources

Smaller attractors with sharper color signatures. They should feel volatile, contaminating, and modular.

Node Type

Design language docs

Docked results that feel collectible. They are stable endpoints, but always forkable back into the field.

Motion

Slow drift, precise response

Ambient motion should imply an active system, while user interactions stay crisp and exact.

Gallery System

Browsing the resulting worlds

The gallery should feel like a design archive with lineage, not a folder of outputs.

Mode 01

Poster wall

High-speed browse for visual scanning. Dense labels, influence chips, and strong typographic titles keep it editorial.

Mode 02

Lineage map

A branch graph reveals which docs share ancestry, which got contaminated, and which mutations produced the strongest departures.

Mode 03

Compare rail

Pin two to four design languages and inspect differences in density, tone, motion rules, and source contamination.

Design Storm should feel like an instrument for steering creative divergence, not a polite AI form with prettier output cards.
