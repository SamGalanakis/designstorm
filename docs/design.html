Principles

Philosophy

Rules governing every decision in the Light Table system.

Core Rules

Content is the brightest thing

Generated designs sit on the lit surface. Everything else is dark chrome. Hierarchy is absolute: work > UI.

Conversation is the interface

No separate gallery. Designs appear inline in the chat. The user talks, the agent makes.

Print production as texture

Crop marks and registration labels are atmosphere, not function. They set tone without demanding attention.

Two surfaces, not gradients

Dark workspace and lit surface. Binary contrast. Cards are on the table or on the light box. No middle ground.

Layout
Single Column

Centered conversation (max 720px). References in a right drawer. Topbar handles navigation. Nothing competes with the chat.

Fixed Anchors

Topbar and composer are fixed. The scroll region between them is the workspace — the light table surface.

Progressive Disclosure

Sessions in a dropdown. References in a drawer. Settings in a sheet. Default view = conversation + designs only.

Compact Spacing

8–14px radii. Tight padding. Professional density. This is a power tool, not a marketing page.

Anti-Patterns
Never: ✕ Rounded pills (999px radius) for structural elements ✕ Box-shadow glow on accent colors (except solid cyan buttons) ✕ Serif fonts (Cormorant Garamond is dead) ✕ Multi-column dashboard layouts ✕ Decorative gradients (only the backlight glow) ✕ Animations over 350ms for UI interactions ✕ Purple. Ever.
