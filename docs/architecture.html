Design Storm / Architecture

Chat-First Design Sessions

The shipped app is a session-based studio: one chat thread, one reference shelf, a persistent async job queue, and a gallery of generated designs. The old board/node graph is gone from the runtime, schema, and primary UX.

Primary UX

Sidebar sessions, central gallery, right-rail chat composer and references. The user talks to one agent and iterates by selecting references or prior designs.

Agent contract

The session agent either replies conversationally or calls make_design exactly once per turn. Design generation is always asynchronous.

Persistence

Sessions, messages, references, jobs, and generated runs are all stored in Postgres. Generated workspaces are previewable from disk and can be recovered from archived ZIPs.

Surface

Server Responsibilities

Session layer
  • Create and rename sessions.
  • Load sidebar, messages, references, jobs, and session-specific runs into one snapshot payload.
  • Persist user and assistant messages, including optional links from assistant messages to queued jobs.
Generation layer
  • Resolve selected reference handles into a frozen JSON snapshot.
  • Create a durable design_jobs record before background work starts.
  • Turn completed jobs into storm_runs with optional iterates_on_id lineage.

Tooling

Session Agent Contract

Custom tool
make_design({ prompt: string, reference_ids?: string[], iterates_on_id?: string, title?: string })
  • The agent is only allowed to use reference handles already selected in the draft.
  • If no reference_ids are passed, the backend defaults to the selected draft references.
  • The backend snapshots those references at job creation time so in-flight work is stable.
  • The tool can only be called once per assistant turn.
Storage

Generated Artifact Lifecycle

Live workspace

Successful generation creates a workspace directory on disk, stores a storm_runs record, and exposes the preview through /preview/{run_id}/.

Archive fallback

If S3-compatible storage is configured, the workspace is zipped and uploaded. Preview and download routes can recover from that archive when the local workspace is missing.

UI

Current Client Model

Gallery
  • Shows pending jobs first, then finished designs.
  • Each design card exposes preview, download, lineage, iterate, and use-as-reference actions.
Composer
  • @ opens mention search over current-session references and prior designs.
  • Shift-click on a design card or reference adds it to the draft selection.
  • An optional iteration target sets the default iterates_on_id for the next turn.
